Kyle built this recipe app in about one afternoon by having a conversation with an AI assistant called Claude. Rather than writing thousands of lines of code by hand, he described what he wanted in plain English, and the AI generated the code. üß† What is AI, Really?

AI, or artificial intelligence, refers to computer systems that can perform tasks that traditionally required human intelligence ‚Äî like reading handwriting, understanding language, or recognizing patterns.

The AI used here (Claude, made by a company called Anthropic) is a large language model. It was trained on billions of pages of text ‚Äî books, articles, code, documents ‚Äî and learned to predict patterns in language. Through this training, it developed the ability to understand context, follow instructions, and generate coherent responses.

A chemistry analogy: Training an AI model is somewhat like optimizing a reaction. You have inputs (training data), conditions (the model architecture), and you iteratively adjust parameters to minimize error ‚Äî similar to how you'd tune temperature, pressure, or catalysts to maximize yield. The "trained model" is the stable, optimized result.

What's remarkable is that these models can now see images. When you upload a photo of your handwritten recipe card, the AI analyzes the pixels, recognizes the letter forms, and extracts the text ‚Äî including your cursive! This capability barely existed three years ago.

üèóÔ∏è How the System Works

Your recipe app has four main components:

  • The Frontend ‚Äî What you see and interact with. It's a website (HTML, CSS, JavaScript) hosted for free on GitHub Pages.
  • The Database ‚Äî Where your recipes are stored. Think of it as a structured spreadsheet in the cloud, with columns for recipe name, ingredients, directions, etc.
  • The Storage Bucket ‚Äî Where the original photos of your recipe cards are kept. Like a folder on the internet.
  • The API Layer ‚Äî The middleman that connects everything. When you upload a photo, the API receives it, sends it to Claude for reading, gets back the extracted text, and saves it to the database.

An API (Application Programming Interface) is simply a defined way for different software systems to communicate. When the website needs to save a recipe, it sends a structured request to the API; the API processes it and sends back a response. It's like a standardized form that both parties agree to use.

A database is structured data storage. Unlike a Word document or a pile of index cards, a database organizes information into tables with defined columns. This makes it fast to search ("show me all cookie recipes") and reliable (data doesn't get corrupted or lost).

üí¨ How Kyle Built This

Kyle used a tool called Cursor, which is a code editor with AI built in. Instead of typing code directly, he typed requests like this:

// Kyle's request:
"Add a PIN screen so only family can access the recipes. Store the PIN securely so it can't be seen in the code."

// AI's response:
[Generates the PIN screen code, creates a secure server function to verify it, updates the database schema]

// Kyle reviews, then:
"Make the PIN input larger and add a 'shake' animation if the wrong code is entered."

[Updates the code accordingly]

This back-and-forth continued for a few hours. Kyle didn't need to know the syntax of every programming language involved ‚Äî he just needed to know what he wanted and how to evaluate whether the result was correct.

This is why it's called "vibe coding" ‚Äî you describe the vibe, the feel, the behavior you want, and the AI handles the technical implementation.

‚ö° Why This is Significant

Three years ago, building an app like this would have required:

  • A professional developer (or team) working for weeks
  • Setting up and maintaining servers
  • Licensing or training an OCR (optical character recognition) system ‚Äî which still probably wouldn't handle cursive well
  • Significant cost ‚Äî easily thousands of dollars

Today, with AI-assisted development and modern cloud services, Kyle built this in an afternoon at minimal cost. The monthly expense to run this app is about $25 ‚Äî less than a dinner out.

More importantly, the cursive handwriting recognition actually works. That's the part that was nearly impossible until very recently. The same AI that can read your handwritten recipe cards can also write code, answer questions, and analyze images ‚Äî capabilities that are genuinely new in the world.
